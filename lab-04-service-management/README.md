# Lab 04 — Service Management with systemd

## Overview

This lab focuses on managing system services in Linux using systemd. 
Systemd is the service manager used by most modern Linux distributions to start, stop, restart, and monitor system services.

## Objectives

- Understand what systemd and services are
- View running services
- Start and stop services
- Restart services
- Enable and disable services at boot
- Check service status

## Key Concepts

A **service** is a background program that runs continuously on a system to provide functionality.

Examples include:
- web servers
- database servers
- ssh server
- networking services

systemd uses the `systemctl` command to manage these services.

## Tools Used

- systemctl
- service status commands
- Linux terminal

## Outcome

By completing this lab, I practiced managing Linux services using systemctl, including starting, stopping, restarting, and enabling services.
