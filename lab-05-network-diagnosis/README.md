# Lab 05 — Network Diagnosis in Linux

## Overview

This lab focuses on diagnosing and troubleshooting network connectivity issues using common Linux networking tools. Network troubleshooting is an essential skill for system administrators, DevOps engineers, and cloud engineers.

## Objectives

- Check network connectivity
- Test communication with remote servers
- Diagnose DNS issues
- Identify network routes
- Inspect open ports and network connections

## Key Concepts

Network diagnosis involves verifying whether a system can communicate with other systems across a network.

Tools like ping, traceroute, nslookup, and netstat help engineers quickly identify where connectivity issues occur.

## Tools Used

- ping
- traceroute
- nslookup
- netstat
- curl
- ip

## Outcome

By completing this lab, I practiced diagnosing network connectivity and DNS resolution issues using standard Linux networking tools.
